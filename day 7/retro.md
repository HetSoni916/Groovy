## Week 1 Retrospective

### ✅ What Worked Well

The first week was a great introduction to the AI-assisted software development workflow and the team environment at Groovy. I successfully set up all the required development tools and AI platforms, including VS Code, Continue.dev/Cline, Codeium, Claude.ai, ChatGPT, Gemini, and the required API accounts. I also completed the initial onboarding activities such as the office tour, team introduction, reading CLAUDE.md and PLAYBOOK.md, interacting with the team on Slack, and sharing my first “Hello AI” experience.

I gained hands-on experience with AI-powered IDEs and learned how to use context effectively by attaching files, folders, and entire codebases to AI assistants. I built a full-stack TODO application using prompts only with React and Node.js, documented all the prompts used in the README, pushed the project to GitHub, and received feedback during the code review session.

The prompt engineering bootcamp helped me understand the importance of writing clear and structured prompts. I practiced converting weak prompts into effective ones, experimented with different prompting styles for the same task, explored few-shot prompting, and created a personal prompt library containing reusable templates that can improve future productivity.

I also explored advanced AI workflows by creating separate Claude Projects for frontend development, backend development, and code review. Comparing ChatGPT, Claude, and Gemini using identical prompts gave me practical insight into their differences in areas such as code generation, reasoning, response quality, and usability.

The highlight of the week was building a student-management CRUD application using React, Node.js, and PostgreSQL with approximately 80% AI-assisted development. I collaborated with a senior developer during the architecture session, completed the implementation independently, demonstrated the project to the cohort, pushed the code to GitHub, and participated in the sprint retrospective.

---

### ⚠️ Challenges Faced

One of the main challenges was learning how to communicate effectively with AI tools. In the beginning, generic prompts produced incomplete or inaccurate responses, which required multiple iterations and better context to achieve the desired results.

Managing AI-generated code was another challenge. Although AI accelerated development, understanding the generated code, debugging issues, and making architectural decisions still required strong problem-solving skills.

While comparing different AI models, I noticed that each model had different strengths and limitations. Some generated cleaner code, some provided better explanations, while others required more precise instructions to produce useful results.

Working with a complete CRUD application in a limited timeframe was also challenging, especially while handling frontend-backend integration, API communication, database operations, and debugging.

---

### 📚 Key Learnings

* AI is most effective when provided with clear requirements, proper context, and well-structured prompts.
* Prompt engineering has a significant impact on the quality of AI-generated output.
* Different LLMs have different strengths, and choosing the right model depends on the task.
* AI-generated code should always be reviewed, tested, and understood rather than accepted blindly.
* Context management inside AI IDEs greatly improves the quality of generated solutions.
* Full-stack application development involves not only coding but also architecture, debugging, testing, and iteration.
* Documentation of prompts and development decisions helps improve collaboration and reproducibility.
